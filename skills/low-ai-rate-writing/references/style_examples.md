## 1. 口语化词汇表

**常用口语词：**
- 搞、跑、装、弄
- 挺、有点、差不多
- 说实话、其实、说白了
- 还行、挺舒服、有点卡

**避免使用：**
- 复杂、完善、优化、提升
- 显著、重要、关键
- 不仅...而且...
- 从...角度来看

## 2. 句式改写示例

**平行结构 → 非平行结构：**

❌ AI式：
```
Gemma 4 采用密集型架构和混合专家架构，支持文本生成、编码和推理等任务。
```

✅ 人类式：
```
Gemma 4 分四个版本。

最小的叫 E2B。23 亿参数。只要 4GB 内存。能看图片，能听音频。手机上都能装。
```

**描述式 → 命令式：**

❌ AI式：
```
用户可以使用以下命令来启动对话。
```

✅ 人类式：
```
然后拉 26B 试试。ollama run gemma4:26b
```

**长句 → 短句：**

❌ AI式：
```
Gemma 4 的上下文窗口最多可容纳 25.6 万个 token，并支持 140 多种语言。
```

✅ 人类式：
```
上下文能到 25.6 万。支持 140 多种语言。
```

## 3. 结构改写示例

**对称列表 → 散文叙述：**

❌ AI式：
```
这个工具主要有三个特点：
1. 支持多种格式
2. 操作简单
3. 性能优秀
```

✅ 人类式：
```
这个工具挺方便的。什么格式都能处理。操作也简单，点几下就搞定。速度还挺快，我用着没卡过。
```

**逻辑连接 → 自然转场：**

❌ AI式：
```
首先，我们需要安装软件。其次，配置环境变量。最后，运行程序。
```

✅ 人类式：
```
先装软件。然后配一下环境变量。完事儿直接跑就行。
```

## 4. AI初稿 vs 人类修改对比

**技术产品介绍：**

❌ AI初稿（65% 疑似 AI）：
```
Gemma 4 采用密集型架构和混合专家 (MoE) 架构，非常适合文本生成、编码和推理等任务。这些模型有四种不同的规模：E2B、E4B、26B A4B 和 31B。这些模型的大小各异，因此可部署在从高端手机到笔记本电脑和服务器的各种环境中。
```

✅ 人类修改（7% 疑似 AI）：
```
Gemma 4 分四个版本。

最小的叫 E2B。23 亿参数。只要 4GB 内存。能看图片，能听音频。手机上都能装。

升级版叫 E4B。45 亿参数。6GB 内存。图片、音频都支持。日常用这个就够了。
```

**修改要点：**
1. 打破平行结构（四种不同 → 分四个版本）
2. 使用命令式（不是"可部署在"，而是直接说规格）
3. 添加口语化表达（"就够了"、"都能装"）
4. 变化句子长度
5. 去除逻辑连接词

## 5. 反模式清单

**绝对避免的模式：**

| 反模式 | 示例 | 改写 |
|--------|------|------|
| 对称结构 | "既...又..." | 直接说两个事实 |
| 学术连接词 | "此外""值得注意的是" | "然后""还有"或直接删掉 |
| 总结性词汇 | "综上所述" | "说白了"或直接删掉 |
| 程度副词 | "显著""重要""关键" | 直接说事实 |
|  hedging | "在一定程度上" | 直接删掉 |
| 长定语 | "采用密集型架构和混合专家架构的" | 拆成短句 |
| 被动语态 | "可以被用于" | "能""可以" |
| 抽象名词 | "性能的提升" | "变快了" |

**标点使用：**

| 避免 | 替代 |
|------|------|
| 冒号":" | 逗号或直接断句 |
| 破折号"——" | 逗号或句号 |
| 分号";" | 句号 |
| 双引号"" | 「」或直接不加 |

## 6. 自检清单

**发布前检查：**

- [ ] 全文没有"此外""值得注意的是""综上所述"
- [ ] 全文没有冒号、破折号、分号
- [ ] 段落长度不均匀（有的1行，有的5行）
- [ ] 有短句（1-3个字）
- [ ] 有第一人称（"我觉得""说实话"）
- [ ] 有口语化词汇（"搞""跑""装""挺"）
- [ ] 有重复用词（真人会重复）
- [ ] 没有"既...又...""不仅...而且..."
- [ ] 没有对称的列表结构
- [ ] 读起来像聊天，不像文章

**快速测试：**
大声朗读一遍，如果听起来自然，就像你在跟朋友聊天，那就对了。如果听起来像在念稿，那就需要改。
