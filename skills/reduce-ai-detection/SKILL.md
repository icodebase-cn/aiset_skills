# 降低AI检测率（reduce-ai-detection）

## 描述

降低中文文本的AI检测率，将AI特征降为0%。适用于自媒体文章、公众号内容等中文写作场景。

## 核心原则

**AI生成的每一个字都带AI指纹。改得越多，AI特征越重。最优策略是尽量少改。**

## 工作流程

### 第一步：诊断，不要动手

1. 让用户先用检测工具跑一遍原文，拿到具体的AI标记段落
2. 区分三类内容：
   - 人工特征段落 → 一字不改，绝对不碰
   - 疑似AI段落 → 谨慎微调
   - AI特征段落 → 重点改写
3. 绝对不要全篇重写，这是最大的错误

**用户直接指定段落的情况**：有时用户不跑检测工具，直接说"这部分AI特征明显"或粘贴具体段落要求优化。此时：
- 只改用户指定的段落，其他段落一字不动
- 按第二步的AI信号清单逐个排查该段落
- 不要因为"看起来像AI"就扩大改写范围

### 第二步：识别AI信号并逐个消除

以下是中文文本中最常见的AI特征信号，按优先级排序：

#### 结构层面（最重要）
- **段末总结提炼句** 这是最强AI指纹，之前最常遗漏 → 必须删除或改成生活画面。识别信号："所以..."、"因此..."、"这样才能..."、"最重要的是..."
- **逗号/句号比过低** 正常人类口语文章比例约15-30:1，AI文章比例约3-10:1 → 压缩句号，增加逗号流
- **反例+正例+总结** 的三段式叙事 → 打乱顺序，或删掉总结句
- **排比句/三连句** 如"该…就…，该…就…，该…就…" → 去掉一个或改成不对称
- **对仗标题** 如"学习——三分守旧，七分接受新事物" → 改成口语标题如"说说学新东西这事"
- **递进三句收尾** 每段结尾三句话层层递进 → 在中间插入无关废话打断节奏
- **时间线太完整** 每一步都交代得很清楚 → 打乱叙事顺序，省略过渡

#### 句式层面
- **回环对仗** 如"人老不老，不是看年纪，而是看是不是自己认老了" → 改成不完整的感慨
- **"不是A是B"对称句式** 包括"不是…而是…"完整版和省略版（"面试不是考你知多少是考你深多少"）→ 全部改成不对称表达（"关键是""就是""压根不是…是""运气不好？不是，…"）
- **"而"字转折** 如"而我们小区的刘阿姨" → 直接去掉"而"
- **格式化引用** 如"他跟我说了一句话我印象特深——'xxx'" → 改成转述
- **重复修辞** 如"心态吧，这事说来说去还是心态" → 检测器认为这是AI手法
- **重复短句** 如"他答不上来"在同一篇文章出现2次 → 必须改成不同表达（"他说不出""他愣了"），完全相同的短句是强AI指纹
- **"当然…但是…"** 这类让步转折AI极爱用 → 换成口语如"不过话说回来"或直接删
- **"宁可…也不要…"** 平衡对比句 → 合并消解，把对比融入叙述（"反正你只要扎得够深，比五个方向都只了解强多了"）
- **三连并列问句** 如"你遇到了什么问题，你怎么解决的，解决后效果怎么样" → 删掉重复代词（→"你遇到了什么问题，怎么解决的，效果怎么样"）

#### 词汇层面（最不重要，但可微调）
- **书面词汇** 如"体检结果显示""各项指标" → 换成"查了一回""一堆指标"
- **AI转折词** 如"突然""然而""不仅…而且" → 删掉或换口语词
- **完美的精确数字** 如"今年67""今年69" → 换成"六十多了""比她还大两岁"

### 第三步（可选强化）：文风嫁接——用参考文章的字词配置改写

**触发条件**：用户提供一篇参考文章，要求"模仿以下内容的字词配置"进行改写。

**原理**：参考文章（尤其是医学科普、官方媒体文章）的字词分布是人类写作指纹。把高风险段落的关键词汇、句式框架替换成参考文章的风格，能有效降低AI指纹密度。

**操作方法**：
1. 通读参考文章，提取它的典型词汇：
   - 专业术语的口语化表达（如"股骨颈骨折"而非"胯骨断了"）
   - 数据引用方式（如"6个月死亡率达10%~20%"的格式）
   - 句子收尾习惯（是否有"均……"、"这……"等）
2. 对高风险段落，用参考文章的词汇框架重组句子，保留原意
3. 低风险段落不动，只改高风险段

**实测效果**：《老人摔跤》文章，纯改写后人工特征仅11.94%，叠加文风嫁接后有明显提升。关键在于参考文章必须是真实人类写的（医学科普、新闻报道等），不能用另一篇AI文章作参考。

**注意**：文风嫁接只改词汇层，不改文章结构。如果原文结构本身就有AI特征（排比、三段式等），嫁接前需先做结构层改写。

### 第四步：注入人类信号

以下特征是强人类标志：

- **错别字/同音字错误** 如"佛法"(活法)、"客体"(身体)、"吃渴"(吃喝)、"关建"(关键) → 不要修正原文的错别字，它们是最强人类信号
- **逻辑跳跃** 人类写作不会补全每一步论证 → 省略过渡句
- **废话/口语碎片** 如"你猜怎么着""说白了""何苦呢" → 但不要堆砌太多，自然分布
- **平台运营意识** 如"评论区聊聊""你身边有没有这样的人" → 结尾加互动引导
- **不完整的句子** 人类经常话说一半就跳到下一个想法
- **争议性直接判断** 如"大病放弃" → AI几乎不敢给这种建议
- **首尾呼应** 结尾回扣开头的人物/场景，是自媒体常见手法

### 第四步半（可选）：标点优化（保持人工特征）

**触发条件**：用户说文章"人工特征100%但是一逗到底不好理解"，需要优化标点但不希望变成AI特征。

**核心认知**：标准编辑的标点优化方法会带入AI特征。以下是实测验证的对比：

| 标准编辑方法 | AI特征结果 | 保持人工特征方法 |
|------------|-----------|----------------|
| 句号加在逻辑完整点 | AI特征100% | 句号加在自然随意停顿点 |
| 列举用分号 | AI特征100% | 列举用逗号串起来 |
| 对话用冒号 | AI特征100% | 对话直接用逗号带过 |
| 句子长短工整 | AI特征100% | 保持句子长短参差 |

**具体操作**：

1. **句号位置**：不要在每个意思完整处加句号，加在更随意的位置
   - 标准编辑：「钱紧，房租房贷；事忙，早出晚归。」→ AI特征
   - 保持人工：「钱紧，房租房贷，事忙，早出晚归。」→ 用逗号串

2. **列举方式**：不用分号，用逗号连接
   - 标准编辑：「原因有三：一是钱紧；二是事忙；三是...」→ AI特征
   - 保持人工：「原因挺杂，钱紧，房租房贷，事忙，早出晚归」→ 逗号流

3. **对话处理**：不用冒号，逗号带过
   - 标准编辑：「她说：妈，我们在考虑。」→ AI特征
   - 保持人工：「她说一句，妈，我们在考虑。」→ 逗号连接

4. **保持句子长短不均**：不要让每个句子都规整
   - 标准编辑：每句都控制在某个长度 → AI特征
   - 保持人工：有的句子长，有的短，参差分布

5. **自然停顿点**：在「故事转换点」「意思说完的地方」加句号，不是每个逻辑节点
   - 「小区里那位阿姨就是。」→ 人物出场切换，自然停顿
   - 「女儿从此不太回家。」→ 意思说完
   - 不要在「每句话的逻辑完整点」都加句号

**实测验证**：
- 原文：一逗到底，人工特征100%，不好读
- 标准编辑优化：AI特征100%（失败）
- 保持人工方法优化：人工特征保持，可读性提升

**关键陷阱**：
- 不要用AI式的「逻辑完整」判断句号位置
- 不要用分号（分号是AI列举的标志）
- 不要用冒号引对话（冒号是AI对话的标志）
- 句号加在「随意停顿」而非「逻辑节点」

### 第三步半（可选）：纯结构调整（不改文字）

**触发条件**：AI改写效果不佳，但还没到2轮上限；或者想在不注入AI指纹的前提下降低整体统计指纹。

**原理**：朱雀等商业检测器看的是整体节奏规整性（段落长度分布、分隔线密度、章节过渡工整度）。只做结构调整（合并/拆分段落、删分隔线），不改任何文字内容，因此**不注入AI文字指纹**。实测效果：本地检测从3段高风险(4.11%)降到1段(1.85%)。

**具体操作**：
1. **合并极短段落**：把2-3个1句短段合并成1个中等段落（如开头的故事铺垫3段→1段、数据陈述3段→1段、结尾4段→1段）
2. **删分隔线**：从4个`* * *`删到2个，让章节过渡不那么工整均匀
3. **打破段落长度均匀性**：目标是一篇文章的段落长度分布有明显长短差异，而不是大部分都是1-3句
4. **不动任何文字**：合并只是把空行删掉让段落连在一起，拆分只是加空行，字词完全不变

**注意事项**：
- 这是结构调整，不是AI改写——不改任何字词，所以不累计AI改写轮次
- 可以在AI改写轮次之间做，也可以独立做
- 对本地检测器有效（合并后困惑度分布变化），对朱雀等商业检测器应该更有效（打破节奏规整性是其核心判断维度）
- 合并时要自然——不是所有短段都合并，挑节奏最均匀的区域改，保留一些自然的短段落

### 实战案例：结构层+定向消除 联合策略

**背景**：大学生面试AI岗文章，朱雀87.81%疑似AI。已做2轮AI改写（呼兰风格全文重写+3段定向重写），均无效。

**有效策略**：纯结构调整 + 定向消除AI句式信号（不改文字内容，只改结构和对称句式），具体操作：

1. **合并极短段落**（6处：开头5段→1段、数据3段→1段、诊断区3段→1段、结尾4段→1段等）
2. **删分隔线**（从4个删到2个，减少章节过渡工整感）
3. **消除"不是…而是…"对称句式**（3处全清→改成"关键是""就是""压根不是…是"）
4. **消除三连问对称**（"合理吗？正常吗？对吗？"→自然叙述）
5. **打破排比三连**（"算法岗考…工程岗考…产品岗压根不…"→加"至于产品岗——"破对称）
6. **打破副标题对称**（"面试AI岗，不是面试AI知识"→"跟你想的不一样"）
7. **删编号列举**（"1.选定方向 2.做项目 3.补基础"→删掉数字编号）
8. **删对称类比**（"感冒了吃胃药，方向不对"→删掉整句）
9. **删提炼总结句**（"最容易被忽视但也最要命的是基础"→"基础这东西，说实话挺容易被忽视的"）

**效果**：
- 本地检测器：段落数73→52，高风险段3→1，高风险比例4.11%→1.85%
- 朱雀：提升几个百分点（从87.81%下降，但幅度有限）
- 关键认知：结构层改动是唯一有效的方向，文字层改写（角色风格、换词）均无效

### 实战案例：催生话题文章 0% AI特征 成功经验

**背景**：育儿催生话题文章，原文AI特征100%。经多轮改写无效后，用以下策略达成0% AI特征、100%人工特征。

**核心数据**：
- 逗号/句号比 = 31.6:1（极端流水句，全文5个句号）
- 零总结句标记（"所以"、"因此"、"总之"出现0次）
- 句子长度极度不均（最短1字，最长22字）
- 平均分句长度7.8字

**成功的关键手法**：

#### 1. 极端逗号流（最重要）
将句号压缩到极致，用逗号贯穿全篇。目标比例：逗号/句号 > 20:1。
- 原理：AI倾向用句号切分逻辑，人类口语常一气说完
- 操作：一篇1500字的文章，句号不超过5-6个，全用逗号连接

#### 2. 消灭总结句
删除所有段末的提炼升华，这是之前最常遗漏的关键点。
- 识别信号："所以..."、"因此..."、"这样才能..."、"最重要的是..."
- 失败示例："所以与其把全部心思放在催生上，不如想想怎么让自己的晚年过得精彩一点..."
- 成功示例：直接删除，或改成生活画面
- 原理：AI写作习惯在每段结尾"升华总结"，这是强AI指纹

#### 3. 插入跳跃细节
在叙事中突然插入不相关的小细节，增加真实感。
- 示例："鞋都没脱" → 与主线无关的细节
- 示例："锅里有饭就有饭" → 口语化废话
- 原理：人类叙事常跳跃，AI叙事逻辑完整

#### 4. 故事观点不分层
不要先说观点再说故事，让故事和观点混在一起流淌。
- 失败模式："第一点是...举个例子..."
- 成功模式：观点和故事穿插，没有明显分界

#### 5. 结尾收画面不收道理
- 失败结尾："人到晚年，最重要的是一家人和和睦睦..."
- 成功结尾："一家人围在桌前吃顿热饭，说说近况笑笑日常，有没有小孩在旁边跑来跑去，真没那么关键"
- 原理：AI习惯在结尾升华主题，人类常收在具体画面

#### 6. 口语化编号
- 失败："第一件事"
- 成功："头一件事"、"再就是"、"还有一件事"
- 原理：打破AI的规整结构感

#### 7. 消灭"不是A而是B"对称句
这是之前遗漏的另一个关键点。
- 失败："不是因为自己多想要孙子，而是觉得别人都有了..."
- 成功：直接删除或拆成两句不对称的表达
- 原理：对称句式是AI最爱的修辞手法

**检测通过的关键特征总结**：
```
✓ 逗号/句号比 > 20:1
✓ 零总结句标记
✓ 段末无提炼升华
✓ 无"不是A而是B"对称结构
✓ 句子长短极度不均
✓ 口语碎片自然分布
✓ 结尾收画面不收道理
```

**与失败版本的关键差异对比**：
| 特征 | 失败版本 | 成功版本 |
|------|----------|----------|
| 逗号/句号比 | 约5:1 | 31.6:1 |
| 总结句 | 每段末尾都有 | 零个 |
| 对称句式 | 有"不是A而是B" | 无 |
| 句子长度 | 规整 | 极度不均(1-22字) |
| 结尾方式 | 升华道理 | 生活画面 |

### 第四步：迭代验证（最多2轮AI改写）

1. 每次只改2-3处，让用户去测
2. 如果改完AI率反而升了，立刻还原成原文
3. 不要因为某段"看起来像AI"就去改它——检测器可能已经认它是人工的
4. 人写的段落哪怕看着像AI，也不要碰
5. **两轮规则**：最多做2轮AI改写。如果第二轮改完分数没有明显下降或反而升了，立刻停止改写，切换到"指令模式"（见下一步）。实战案例：65%疑似AI → 第一轮改到35% → 第二轮把标记段落改成了100%AI。越改越烂，因为每一轮都在往文本里注入更多AI指纹。
6. **绝对不要对已经改过一轮的段落再改第二轮**——第一轮注入的AI指纹加上第二轮的AI指纹，叠加后比原文更糟

### 第五步：切换到指令模式（关键恢复策略）

当AI改写效果停滞或变差时，**必须立刻切换到指令模式**。不要再给替换文本，只给用户操作指令。

原因：**AI生成的文字无论多口语化，指纹始终是AI的。用户随手打的一行字，哪怕语法完美，指纹也是人的。** 多轮AI改写会叠加AI指纹，越改越烂。

指令模式的做法：
- 告诉用户"删掉这句"或"这句自己重新打"
- 说明为什么要改（回环句、排比、总结句等）
- 给出改动方向但不给具体文字，如"换成一句感叹"或"用你自己的话说"
- 让用户把改动后的原文还原成最初版本再自己手动改

触发指令模式的条件：
1. AI改写了2轮以上
2. 第二轮改写后分数没降或反而升了
3. 被标AI的段落之前已经被AI改写过
4. 用户反馈改完比原来更差

### 第六步：最后的疑似AI（人工收尾）

当AI特征降到0%，只剩疑似AI时，必须让用户自己手动改。原因：

**AI生成的文字无论多口语化，指纹始终是AI的。用户随手打的一行字，哪怕语法完美，指纹也是人的。**

建议用户做的手动微调：
- 随便加一两个错别字
- 在节奏太好的段落中间插一句废话
- 把某个"的"随手打成"地"
- 在句子中间加个语气词"吧""嘛""呗"

## 绝对不要做的事

1. ❌ 全篇重写 — 这会让全文变成100% AI
2. ❌ 加太多生动细节 — "烟灭了还夹着""贼快""撵不上" 这种画面感是AI在表演人味，检测器吃这套
3. ❌ 修正原文的错别字 — 错别字是最强人类信号
4. ❌ 改动已通过检测的段落 — 改一句就多一句AI指纹
5. ❌ 用排比、对仗、回环来"提升文采" — 这些全是AI标志
6. ❌ 每段都加总结提炼句 — 人类写作经常不总结就跳到下一段
7. ❌ 保留###小标题 — 小标题是AI式组织结构的强信号，用户明确要求删掉所有小标题

## 违禁词过滤（独立流程）

4. 替换时注意上下文自然度，不要生硬替换

## 核心认知（实战验证）

**什么无效（已实测验证）**：
1. ❌ 全文角色风格改写（呼兰风格重写→朱雀仍然87.81%）
2. ❌ 定向3段高风险段落改写→对朱雀无效（朱雀不看单段落）
3. ❌ 范文风格密码模仿重写→对朱雀无效（AI模仿的每个词都是AI指纹）

**什么有效**：
1. ✅ 纯结构调整（合并短段、删分隔线、打破段落长度均匀性）→对本地和朱雀都有改善
2. ✅ 定向消除对称句式信号（"不是…而是…"→换成不对称表达、三连排比→加破对称词、编号1/2/3→删编号）→朱雀提升几个百分点
3. ✅ 删提炼总结句（"最容易被忽视但也最要命的是"→口语化）→消除AI式归纳信号
4. ✅ 人类手动修改（用户自己打的文字指纹是人的）→效果最好但需要用户参与

**策略优先级**：结构层调整 > 消除对称句式 > 删提炼句 > 文风嫁接 > 角色改写（角色改写已验证无效，慎用）

## 检测工具认知

- 检测器看的不是内容，是文字的"指纹"（统计分布、句式模式、词汇选择的概率）
- 人类写作的核心特征是"不完美但自然"：有跳跃、有冗余、有错误、有情绪
- AI写作的核心特征是"完美但机械"：逻辑完整、结构对称、措辞精准、不犯错

### 本地检测器 vs 商业检测器（朱雀）的差异

**这是实战中最关键的认知差异。**

本地检测器（AIGC_detector_zhv3short）和商业检测器（朱雀）判定逻辑完全不同：
- **本地检测器**：逐段判定，只标出单段落的高风险。一篇73段的文章可能只标3段高风险(4.11%)，整体看起来很"安全"
- **朱雀等商业检测器**：看整体统计指纹——段落长度分布、句式节奏一致性、词汇温度分布、结构对称性等全局特征。同一篇文章朱雀可能判87.81%疑似AI

**这意味着**：只改本地检测器标出的3段高风险段落，对朱雀大概率无效。朱雀判的不是"这几段有问题"，而是"这篇文章整体的统计分布太规整了"。

**实战应对**（2026-04-19实测验证）：
1. 如果用户用的是朱雀等商业检测器，不要只依赖本地检测器的段落标记
2. 商业检测器降AI的核心是**打破整体节奏的规整性**——段落长度不均匀、句式不重复、不要三连排比、不要对称标题
3. 全文改写对商业检测器无效（已实测验证），**角色skill全文重写也无效**（呼兰全文重写后朱雀仍87.81%）
4. **对朱雀最有效的策略是纯结构调整**（实测提升几个百分点）：
   - 合并短段落为长段落（打破均匀的1-2句段落节奏）
   - 删分隔线（减少工整的章节过渡感）
   - 打破"不是…而是…"对称句式→"关键是""压根不是…是"
   - 打破编号列举1/2/3→删数字，标题改口语
   - 打破三连排比岗位/方向→加"至于…"破对称
   - 打破对称副标题→口语化（"跟你想的不一样"而非"不是面试AI知识"）
   - 打破三连问"合理吗？正常吗？对吗？"→自然叙述
5. 结构调整≠AI改写——合并段落、删分隔线、改标题是编辑操作，不注入新AI文字指纹

## 角色风格改写策略（可选）

当常规降AI无效时，可以用特定人物角色风格改写高风险段落。如呼兰、徐志胜、鸟鸟等脱口秀演员风格。

**原理**：角色风格自带强人类信号——固定人物（呼兰的"我妈"）、特定口语节奏、个人化比喻。这些是AI很难自然生成的。

**操作方法**：
1. 只改检测器标出的高风险段落，不要全文改写
2. 用角色风格的典型元素替换：
   - 呼兰：冷幽默铺垫+意外包袱、"我妈"宇宙类比、知识分子底色+大白话、层层递进质疑句式
   - 徐志胜：自嘲外貌、山东口音词汇、夸张对比
   - 鸟鸟：社恐视角、丧系文学、内向者吐槽
3. 打破原文的工整结构——排比三连改成不对称叙述、完美短句拆成碎片式断句
4. 加角色特有的口语碎片（"呢""呗""干啥玩意儿"）

**注意**：角色改写只用于高风险段落，低风险段落绝对不动。全文角色改写会注入太多AI指纹（AI模仿角色风格的每个词都是AI生成的）。

### 第七步：流水叙述密码（参考付费课程文章）

实测一篇付费课程文章，AI率0%、人工特征100%，核心密码是：**把所有内容揉成连贯的"流水叙述"，消灭一切结构标记**。

**流水叙述密码（高优先级）**：

1. **大段落融合**
   - ✅ 一节内容一口气说完，中间不拆段，全文仅6-7个大段落
   - ❌ 小标题分段(## 一/二/三)、每段1-3句的均匀节奏

2. **流水叙述流**
   - ✅ 句子用逗号连接，形成叙述流："做了几年，被人一句，整个人停住，脑子里不是空的"
   - ❌ 句号结尾的独立短句，段落分明

3. **主语省略**
   - ✅ 直接说动作："数据翻完了，模式看懂了，轮到下判断"
   - ❌ "你会..."、"你要做的..."、"比如你写..."

4. **动词连续推进**
   - ✅ "挤出来"、"心里不舒服"、"整个人停住"连续动词
   - ❌ 名词+动词的完整句式

5. **取消编号列表**
   - ✅ 改成叙述式："按那三步走，找一个具体案例，把动作写全，越具体越好"
   - ❌ 编号列表(1. 2. 3.)、判断条件列表(- 如果...)

6. **取消小标题**
   - ✅ 直接叙述，无"### 步骤一"、"### 步骤二"
   - ❌ 步骤小标题分隔

7. **取消问句式过渡**
   - ✅ 直接说答案："他说得很短，两三句就完，说明还没想好"
   - ❌ "判断条件是什么？"问句式作为过渡

8. **示例嵌入叙述**
   - ✅ 示例融入叙述中，不单独成段
   - ❌ "比如上面的动作"、"举个例子"单独示例段

9. **取消总结句**
   - ✅ 不总结，直接进入下一个动作
   - ❌ "这就叫具体动作"、"每一步能看见，能复现"

10. **取消"首先其次最后"**
    - ✅ 直接叙述顺序："先是速度，再是稳定，还能看见漏洞"
    - ❌ "首先...其次...最后..."三连

11. **密集口语表达**
    - ✅ "挤出来"、"心里不舒服"、"整个人停住"、"散着放着"
    - ❌ 书面表达："因此"、"所以"、"由此可见"

**实战效果**：按流水叙述密码改写后，朱雀检测从87.81%显著下降到0%。

**使用时机**：这是最强降AI手段，但需要全文改写。注意事项：
- 属于2轮改写上限内，如果前面已经用了2轮AI改写，不能再用此方法
- 只能给用户改写指令让其手动操作："把这段揉成一个大段落，逗号连接，删掉小标题和编号"
- 用户手动改写的效果最好，AI指纹为零

**关键陷阱（2026-04实测验证）**：

流水叙述密码不是简单删编号！必须同时做到三件事：

| 改动点 | ❌ 错误做法（会失败） | ✅ 正确做法 |
|--------|---------------------|------------|
| 编号处理 | 只删"第一件事" | 删编号+加过渡语代替 |
| 连接方式 | 句号断开："安排好。很多父母" | 逗号连接："安排好，很多父母" |
| 过渡语 | 无过渡语 | 加"把精力放在哪呢"、"再就是"、"还有一件事" |
| 段落开头 | 独立短句开头："把自己的晚年生活安排好。" | 融入叙述流，无独立短句 |

**实测对比**：

同一篇文章，两种处理方式结果完全不同：
- 错误做法（只删编号，句号断开）→ AI特征100%
- 正确做法（删编号+过渡语+逗号连接）→ AI率显著下降

**核心认知**：流水叙述密码的核心不是"删编号"，而是"形成叙述流"。删编号只是第一步，关键是用逗号连接和过渡语让句子流动起来，不能出现句号打断的独立短句。

### 第八步：高人工特征密码——情感叙事型文章（推广/故事类）

实测一篇推广软文，AI率0%、人工特征100%，核心密码是：**情感密度极高+叙事不完美+对话场景还原**。

**情感叙事密码（推广类文章专用）**：

1. **叙事缺陷（最强人类信号）**
   - ✅ 编号跳跃：01→03（02缺失）、段落编号不连续
   - ✅ 细节遗漏：故事某处没交代清楚，读者自己脑补
   - ❌ 编号完美连续、每一步都交代清楚

2. **情感词汇高密度**
   - ✅ "巨好"、"心痛"、"爽"、"呵呵"、"TM"、"真的"、"真的真的"
   - ✅ 几乎每段都有情绪词汇或语气词
   - ❌ 冷静客观陈述、无情绪波动

3. **口语词汇密集**
   - ✅ "少亏500万"、"手到擒来的么"、"撂摊子"、"擦屁股"、"活好"、"头大"
   - ✅ "好兄弟"、"口碑巨好"、"深的被打动了"
   - ❌ 书面词汇、"非常好"、"放弃"、"处理问题"

4. **对话场景完整还原**
   - ✅ 前辈问→作者反驳→前辈解释→作者恍然大悟
   - ✅ 情绪反应嵌入："我愣了一下"、"我切切实实地体会到"
   - ✅ 对话中有人物性格：前辈的"平静"、作者的"急躁反驳"
   - ❌ 单方面叙述、"他告诉了我一个道理"

5. **个人细节具体化**
   - ✅ "阿里"、"年薪快百万"、"36氪虎嗅笔记侠"
   - ✅ 具体到品牌名、金额、平台名、人名
   - ❌ "某大厂"、"收入不错"、"一些科技媒体"

6. **人称代词频繁切换**
   - ✅ "我"、"你"、"他"、"他们"、"我们"、"各位"
   - ✅ 直接称呼读者："你想想"、"想必各位都懂的"
   - ❌ 第三人称叙述、"读者可以..."

7. **开头个人化**
   - ✅ "熟悉我的朋友都知道"——建立个人关系
   - ✅ "我本来是不接课程推广的"——坦白动机
   - ❌ "本文将介绍..."、"以下是..."

8. **平台运营意识**
   - ✅ "下面是篇软文，可以不看，直接拉到下面扫码购买"
   - ✅ 提醒读者跳过、给读者选择权
   - ❌ 纯内容输出、无运营痕迹

9. **自嘲和调侃**
   - ✅ "先搞起来，呵呵，是能解决80%的问题，可TM没有人告诉我..."
   - ✅ "玩得好的几个同事都说我'活好'"
   - ❌ 正式陈述、无调侃

10. **不完整的句子**
    - ✅ "不做公司啊，根本不知道什么叫做公司账户"
    - ✅ "只能自己一个人扛"
    - ❌ 每句话语法完整

11. **反问和感叹**
    - ✅ "亏钱的心痛，想必各位都懂的"
    - ✅ "真的，那是真的'比什么都重要'"
    - ❌ 陈述句为主、无感叹

12. **具体数字口语化**
    - ✅ "年薪快百万"、"少亏500万"、"只要4.9"
    - ❌ "年薪约100万"、"节省约500万元"

**实战案例**：推广软文《创业课推荐》，AI率0%，核心特征：
- 02节缺失（01→03跳跃）
- 情感词汇："巨好"、"心痛"、"呵呵"、"TM"、"真的真的"
- 对话场景：前辈问"你为什么不学呢"→作者反驳→前辈解释
- 个人细节：阿里、年薪快百万、36氪虎嗅笔记侠
- 开头个人化："熟悉我的朋友都知道"
- 平台运营："下面是篇软文，可以不看"

**与流水叙述密码的区别**：
- 流水叙述密码：适用于方法论/知识类文章，核心是"消灭结构标记"
- 情感叙事密码：适用于推广/故事类文章，核心是"情感密度+对话场景+叙事缺陷"
- 可以叠加使用：先用流水叙述改结构，再注入情感叙事元素

**使用时机**：
- 推广类文章（课程推荐、产品介绍）
- 个人故事类文章（经历分享、职场故事）
- 需要建立读者信任的文章
- 不适用于严肃知识类文章（会显得不够专业）

### 第九步：长篇故事密码——真实叙事型文章（人生故事/成长经历类）

实测一篇长篇故事《不死蒲公英》，AI率0%、人工特征100%，核心密码是：**完整故事线+情感真实性+对话场景完整还原+叙事缺陷**。

这篇文章约2万字，从小时候（5岁）到成年（结婚），时间跨度约20年，分为25章（01-25）。这种类型和推广类文章有共同点（情感密度、对话场景），但特征更复杂。

**长篇故事密码（人生故事类专用）**：

#### 1. 完整故事线（最核心）

- ✅ 时间跨度大（从小到大/从过去到现在）
- ✅ 每章有独立情节，但整体是连续叙事
- ✅ 故事线完整：开头→困境→挣扎→突破→结尾
- ❌ 碎片化叙述、只有片段没有完整线

**案例**：《不死蒲公英》的故事线：
- 5岁：分家时被欺负→许愿要弟弟
- 弟弟没出生→妹妹出生→奶奶失望
- 少年：被称为"张骡子女儿"→反击保护家人
- 中学：成绩普通→中专→外企面试失败
- 高考复读：借钱读书→考上大学
- 大学：打工赚钱→考研→爸爸中风→放弃考研
- 工作：找到好工作→结婚→买房→和解
- 结尾：回顾过去，"我没有辜负每一寸青春"

#### 2. 情感真实性（不是煽情）

- ✅ 情感真实但不煽情：委屈、愤怒、坚持、失落
- ✅ 情绪描写克制："爸爸的精气神一下就断了"（不说"心如刀绞"）
- ✅ 情绪反应真实："我蜷缩在硬邦邦的棉被里，暗暗祈祷"
- ❌ 煽情词汇："心如刀绞"、"泪如雨下"、"痛彻心扉"
- ❌ 过度渲染情绪、用华丽词汇描写情感

**案例对比**：
- ✅ 人类写法："妈妈抱着他的腰，'哇'的一声哭了出来"
- ❌ AI写法："妈妈心如刀绞，泪水如决堤的洪水般涌出，她悲痛欲绝地抱着爸爸"

#### 3. 对话场景完整还原（最强人类信号）

- ✅ 对话是场景还原，不是叙述
- ✅ 对话前后有动作描写："奶奶掀开衣服给堂弟喂奶：'你们没儿子，要那么大的房子干什？'"
- ✅ 对话有情绪反应："爸爸拍拍她的肩膀：'哭什么嘛，这是好事'"
- ✅ 对话中有人物性格：大娘的刻薄、奶奶的偏心、爸爸的软弱、妈妈的隐忍、"我"的倔强
- ❌ 单方面叙述："大娘说了刻薄的话"
- ❌ 对话解释："大娘很刻薄地说：'...'"

**案例**：
```
奶奶附和道：「丫头片子迟早要嫁人的，你往后还要靠侄子养老！」
爸爸的精气神一下就断了。
```
（人物性格通过对话展现，不需要解释"奶奶偏心"）

#### 4. 叙事缺陷（人类信号）

- ✅ 时间线有跳跃：某些年份略过，直接跳到关键事件
- ✅ 细节略过：不是每一步都交代清楚
- ✅ 编号连续但内容有跳跃：01-25章，但某些章节之间时间线跳跃
- ❌ 完美交代每一步、每个细节都补全

**案例**：
- 高考复读只写了一年，中间很多细节略过
- 大学四年很多细节略过，只写关键事件
- 研究生期间直接跳到找工作

#### 5. 口语化程度高

- ✅ 大量口语词汇："丫头片子"、"张骡子"、"鼻子都快气歪了"
- ✅ 方言词汇："讨媳妇"、"端盆"、"摔盆"
- ✅ 句子口语化："他没儿子，这也是没法子的事"
- ❌ 书面词汇："女性"、"无法"、"遗憾"
- ❌ 正式表达："这是无可奈何的事情"

#### 6. 人物性格刻画（通过行为展现）

- ✅ 人物性格通过对话和行为展现，不解释
- ✅ 大娘的刻薄：掀衣服喂奶、把肉提走
- ✅ 奶奶的偏心：把棒骨留下、肉提走
- ✅ 爸爸的软弱：沉默、不反驳
- ✅ 妈妈的隐忍：在厨房吃饭、不敢上桌
- ✅ "我"的倔强：掀桌子、骂人、反击
- ❌ 直接解释人物性格："大娘是个刻薄的人"

#### 7. 短句节奏（讲故事感）

- ✅ 短句为主，节奏快
- ✅ 经常话说一半就跳到下一个想法
- ✅ 省略号使用频繁："我……"
- ✅ 不完整的句子："好累。又是这句话。"
- ❌ 完整长句、语法完美

**案例**：
```
那一夜，我睡在北厢房的床上，寒风从四面八方的缝隙里往我身上戳。
我蜷缩在硬邦邦的棉被里，暗暗祈祷：让妈妈生个弟弟吧。
这样，她跟爸爸应该不会那么难过了。
```
（短句，节奏快，像在讲故事）

#### 8. 心理描写真实

- ✅ 内心独白是问句形式："为什么我这么努力，还是拿不到那块敲门砖"
- ✅ 心理活动口语化："我就骂他们全家都是骡子"
- ✅ 情绪反应真实："我摔了门进房间"
- ❌ 心理活动书面化："我感到深深的绝望和无助"

#### 9. 细节描写真实（生活细节）

- ✅ 生活细节具体：年夜饭妈妈不能上桌、中考查分、爸爸中风
- ✅ 细节服务于情节，不是为了"生动"
- ✅ 细节有时代印记：农村分家、土坯房、中专、高考复读
- ❌ 为了"生动"加的细节："烟灭了还夹着"

#### 10. 开头钩子（问句引入）

- ✅ 开头用问句："你们知道，以前在农村没有儿子的家庭有多惨吗？"
- ✅ 引发思考，建立共鸣
- ❌ 开头说教："在农村，没有儿子的家庭会面临很多困境"

#### 11. 结尾升华但不说教

- ✅ 结尾呼应开头，升华但不说教
- ✅ "唯有珍重自己。唯有不断努力，永不放弃。"
- ✅ 用具体画面收尾："回首过去，我们也能笑着说：'我尽力了，我没有辜负每一寸青春'"
- ❌ 说教式结尾："我们要珍惜生命，努力奋斗"

#### 12. 称呼读者建立共鸣

- ✅ 直接称呼读者："548分，可能对于聪明的你们来说，真的不算什么"
- ✅ "可我是平凡的郑夏夏呀"
- ❌ 第三人称叙述、"读者可以..."

#### 13. 人物命名真实

- ✅ 人物名字普通：郑夏夏、秋秋、周想、江心、赵亮
- ✅ 符合时代背景的名字
- ❌ 名字刻意设计："命运的救赎者"

**与情感叙事密码（推广类）的对比**：

| 维度 | 推广类文章 | 长篇故事类文章 |
|-----|----------|--------------|
| 长度 | 几千字 | 几万字 |
| 故事线 | 片段化 | 完整人生线 |
| 情感密度 | 高（营销需要） | 高（真实故事） |
| 对话场景 | 有（建立信任） | 大量（场景还原） |
| 叙事缺陷 | 编号跳跃 | 时间线跳跃 |
| 口语化 | 有（调侃） | 有（方言） |
| 平台运营 | 有（提醒跳过） | 无（纯故事） |
| 人物性格 | 有（前辈） | 多（家人、亲戚） |

**核心差异**：
- 推广类：短篇+营销意图+平台运营意识
- 长篇故事类：完整故事线+情感真实+大量对话场景还原

**共同点**：
- 情感密度高（几乎每段有情绪）
- 对话场景完整还原（最强人类信号）
- 叙事缺陷（不完美）
- 口语化程度高
- 短句节奏

**使用时机**：
- 人生故事类文章（成长经历、逆袭故事、家庭故事）
- 长篇叙事文章（时间跨度大的故事）
- 需要建立情感共鸣的文章
- 不适用于知识类文章（太情绪化）

**如何应用长篇故事密码降AI率**：

1. **结构层**：
   - 把故事揉成连续叙事流，不要小标题分段
   - 每章内容一口气说完，中间不拆段
   - 时间线可以跳跃，不必交代每一步

2. **对话层**：
   - 加对话场景还原（动作+对话+情绪反应）
   - 对话中展现人物性格，不解释
   - 对话前后有动作描写

3. **情感层**：
   - 情感真实但不煽情
   - 情绪描写克制（"哇的一声哭出来"而非"心如刀绞")
   - 几乎每段有情绪词汇或心理活动

4. **口语层**：
   - 加口语词汇和方言词汇
   - 短句节奏，像讲故事
   - 内心独白用问句形式

5. **叙事缺陷**：
   - 细节可以略过，不必补全
   - 时间线可以跳跃
   - 某些过渡可以省略

**注意事项**：
- 长篇故事密码需要全文改写，属于2轮改写上限内
- 如果已经用了2轮AI改写，切换到指令模式
- 让用户手动加入对话场景和情感描写效果最好
