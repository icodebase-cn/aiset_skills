# 文本处理技巧 (Text Processing Skills)

## 概述
文本处理技巧涉及对小说文本的扩写、总结、重构等操作，帮助作者优化内容表达、调整篇幅和风格。

---

## 1. 文本扩写 (Text Expansion)

### 技巧描述
在保持原文核心情节的基础上，增加细节描述和情感表达，让场景更加生动丰富。

### 核心能力
- **细节丰富**：增加环境、动作、心理等细节描写
- **情节扩展**：在不偏离主线的前提下合理扩展
- **角色深化**：深入刻画角色的心理活动和行为细节
- **环境渲染**：增强场景描写和氛围营造
- **对话优化**：丰富对话内容，增加语言层次

### 扩写原则
- 保持原文的核心情节和人物关系
- 严格按照指定的长度和风格要求执行
- 确保扩写内容与原文风格保持一致
- 让情节发展更加自然流畅
- 避免偏离原文的主要情节线
- 保持故事的连贯性和角色性格的一致性

### 扩写维度
1. **感官细节**：增加视觉、听觉、嗅觉等感官描写
2. **心理活动**：添加角色的内心独白和情感变化
3. **动作细节**：细化动作描写，增强画面感
4. **环境氛围**：扩展环境描写，营造氛围
5. **对话延伸**：丰富对话内容，增加层次感

### 提示词结构
```
## 扩写要求
- **扩写长度**: {{length}}
- **扩写风格**: {{style}}
- **具体指令**: {{instructions}}

## 需要扩写的文本
{{input}}

## 相关上下文
{{context}}
```

## 2. 文本总结 (Text Summary)

### 技巧描述
提炼和总结故事要点，提取关键情节和重要信息，去除冗余细节。

### 核心能力
- **内容提炼**：提取关键情节和重要信息
- **逻辑梳理**：保持总结的准确性和完整性
- **重点突出**：识别并突出重要的故事转折点
- **主题把握**：概括主要主题和情感线索
- **结构优化**：按照要求的详细程度进行总结

### 总结原则
- 准确反映原文的主要内容和情节发展
- 严格按照指定的长度和风格要求执行
- 保持逻辑清晰，条理分明
- 突出关键的情节转折和角色发展
- 保留重要的情感节点和主题元素
- 使用简洁明了的语言表达

### 总结类型
1. **情节总结**：按时间线梳理主要事件
2. **人物总结**：概括角色的发展和变化
3. **主题总结**：提炼作品的核心主题
4. **章节总结**：概括每章的核心内容

### 输出格式
```
## 总结内容
[简洁准确的总结文本]

## 关键要点
- 要点1
- 要点2
- 要点3
```

## 3. 文本重构 (Text Refactor)

### 技巧描述
优化和重构小说文本，改善表达方式，调整语言风格。

### 核心能力
- **文字优化**：改善表达方式，使文字更流畅生动
- **风格调整**：根据要求调整语言风格、叙述角度
- **结构重组**：优化句式结构，改善段落组织
- **细节完善**：补充必要细节，删减冗余内容

### 重构原则
- 保持原文的核心内容、情节发展和人物性格
- 确保与小说整体风格和背景设定保持一致
- 根据上下文信息调整表达方式，保证连贯性
- 尊重作者的创作意图，在此基础上优化
- 严格按照指定的重构方式和长度要求执行

### 重构类型
1. **风格转换**：如从华丽转为简洁，从现代转为古风
2. **视角转换**：如从第一人称转为第三人称
3. **节奏调整**：调整叙述节奏，加快或放慢
4. **情感调整**：调整情感基调，如从悲伤转为希望

---

## 4. 章节大纲提取 (Outline Extraction)

### 技巧描述
从小说文本中提取章节大纲，分析每章的核心剧情。

### 核心能力
- **情节识别**：识别章节的核心情节线
- **事件提取**：提取关键事件和转折点
- **人物追踪**：概括人物发展轨迹
- **设定标注**：标注重要的设定信息

### 提取原则
- 直接按顺序总结每章的核心剧情
- 简洁明了，不需要过多解释
- 突出情节的主线和关键转折
- 保持章节间的逻辑连贯性

### 输出格式
```
【剧情1】[第一章核心剧情]
【剧情2】[第二章核心剧情]
【剧情3】[第三章核心剧情]
...
```

---

## 5. 场景摘要生成 (Scene Summarization)

### 技巧描述
为小说场景生成简洁准确的摘要，保留核心信息。

### 核心能力
- **关键提取**：识别场景中的核心情节点
- **人物把握**：提取主要角色的关键行为
- **环境概括**：总结环境设定和氛围特点
- **情感捕捉**：概括情感变化和心理转折

### 摘要原则
- 准确捕捉场景的核心内容和主要事件
- 严格按照指定的长度和风格要求执行
- 突出关键角色的重要行为和决定
- 简洁明了，避免冗余和次要细节
- 保留推动故事发展的关键信息
- 体现场景的情感基调和氛围

---

## 6. 摘要扩展为场景 (Summary to Scene)

### 技巧描述
将简洁的情节摘要扩展为生动详细的场景描写。

### 核心能力
- **情节还原**：根据摘要构建完整的场景情节
- **细节创造**：创造丰富的环境描写和氛围
- **对话设计**：设计自然流畅的人物对话
- **心理刻画**：添加恰当的心理描写
- **风格统一**：确保场景与小说整体风格一致

### 扩展原则
- 严格遵循摘要中的核心情节和关键事件
- 严格按照指定的长度和风格要求执行
- 合理扩展细节但不偏离主要故事线
- 创造符合小说风格和时代背景的描写
- 确保人物行为和对话符合其性格特征
- 平衡动作、对话、心理和环境描写

---

## 7. 拆细纲 (Detailed Outline Deconstruction)

### 技巧描述
将章节内容拆解为详细的细纲，按顺序列出每个核心剧情点。

### 核心能力
- **剧情拆解**：将复杂情节拆解为独立剧情点
- **顺序梳理**：按时间顺序排列剧情
- **要点提炼**：提取每个剧情的核心要素
- **逻辑梳理**：保持剧情的逻辑连贯性

### 输出格式
```
【剧情1】[第一个核心剧情点]
【剧情2】[第二个核心剧情点]
【剧情3】[第三个核心剧情点]
...
```

---

## 文本处理通用原则

1. **保持核心**：处理后的文本必须保留原文的核心内容
2. **风格一致**：处理结果要与原文风格保持一致
3. **逻辑连贯**：确保处理后的文本逻辑清晰连贯
4. **尊重原意**：在不改变原意的基础上进行优化
5. **灵活调整**：根据具体要求灵活调整处理方式

---

## 应用场景

- **篇幅调整**：根据需要对文本进行扩写或精简
- **风格统一**：调整不同章节的风格使其统一
- **内容优化**：改善表达，提升文本质量
- **大纲规划**：从已有内容提取大纲或扩展大纲
- **快速预览**：通过摘要快速了解内容

## 第四步：四层自检体系

文本处理完成后必须跑完整个四层质检流程。这个自检体系的设计理念来自软件工程中的测试金字塔，从最基础的硬性规则到最主观的活人感判断，层层递进。每一层都有明确的通过标准和修复指引。只有四层全部通过，处理结果才算达标。

### L1 硬性规则检查（自动扫描层）

这一层检查的是绝对不能违反的规则，类似代码的语法检查。任何一项不通过就必须修复，没有例外。

**L1-1 核心内容保留检查**
- 扩写/精简后是否保留了原文的核心情节？
- 总结是否准确反映了原文的主要内容？
- 重构后是否保持了原文的核心信息？

**L1-2 风格一致性检查**
- 处理后的文本风格是否与原文一致？
- 扩写内容是否与原文风格匹配？
- 不同章节的风格是否统一？

**L1-3 逻辑连贯性检查**
- 处理后的文本逻辑是否清晰连贯？
- 情节发展是否自然流畅？
- 段落间过渡是否顺畅？

**L1-4 格式规范检查**
- 是否遵循了指定的输出格式？
- 标记符号使用是否正确？
- 结构层次是否清晰？

**通过标准**：以上四项检查全部通过。
**修复方式**：逐条检查，对不符合要求的处理结果进行修正。

### L2 处理质量检查（模式匹配层）

这一层检查文本处理的质量，类似代码的单元测试。每一项都给出"是/否"判断。

**L2-1 扩写质量检查**（如适用）
- 扩写内容是否丰富了原文？
- 新增内容是否符合扩写要求？
- 是否保持了与原文的连贯性？

**L2-2 总结质量检查**（如适用）
- 总结是否准确概括了原文要点？
- 是否突出了关键情节和转折点？
- 长度是否符合要求？

**L2-3 重构质量检查**（如适用）
- 重构后的结构是否更清晰？
- 是否改善了原文的问题？
- 是否保持了故事的完整性？

**L2-4 大纲质量检查**（如适用）
- 大纲是否覆盖了所有重要内容？
- 层级结构是否合理？
- 是否便于后续创作参考？

**通过标准**：根据处理类型，相关检查项全部通过。
**修复方式**：针对具体问题进行修正。

### L3 内容质量检查（深度审查层）

这一层检查处理后内容本身的质量，类似代码的集成测试。

**L3-1 准确性检查**
- 处理后的内容是否准确反映了原文？
- 是否有误解或曲解原文的地方？
- 信息是否有遗漏或错误？

**L3-2 完整性检查**
- 内容是否完整？
- 是否有遗漏的重要信息？
- 结构是否完整？

**L3-3 可读性检查**
- 处理后的文本是否易读？
- 语言是否流畅自然？
- 是否符合目标读者的阅读习惯？

**L3-4 实用性检查**
- 处理结果是否满足用户需求？
- 是否便于后续使用？
- 是否达到了预期目的？

**通过标准**：L3-1和L3-2必须全部通过，L3-3和L3-4至少通过一项。
**修复方式**：需要重新审视处理结果，补充遗漏、修正错误、提升可读性。

### L4 活人感终审（最终人格层）

这是最重要也是最主观的一层。这一层不是逐项检查，而是以用户的视角审视处理结果，回答一个核心问题：

**"这个处理结果是否像专业人士完成的，还是像AI机械处理的？"**

具体的感知维度：

**L4-1 自然度**
- 处理后的文本是否自然流畅？
- 是否有生硬或机械的痕迹？
- 是否保持了"人味"？

**L4-2 专业度**
- 处理结果是否体现出专业水准？
- 是否符合行业标准？
- 是否让用户感到满意？

**L4-3 价值感**
- 处理结果是否为用户创造了价值？
- 是否帮助用户更好地理解或使用原文？
- 是否值得用户信赖？

**L4-4 心流检查**
- 阅读处理结果时是否顺畅？
- 有没有让人困惑或出戏的地方？
- 如果有，那个地方就是需要修复的问题。

**通过标准**：L4-1到L4-4整体感觉"这像是专业人士处理的"。如果任何一项让你觉得"这段AI味太重了"，就需要返工。
**修复方式**：这一层没有机械的修复方法。核心操作是：把"AI味重"的部分找出来，用更自然、更专业、更有价值的方式重新处理。

完成四层自检后，输出一份简洁的质检报告：
